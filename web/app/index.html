CLI commands

Every command and flag available in Skopix
skopix run
Run a single QA session against a URL with a goal in plain English.
skopix run \
  --url "https://app.com" \
  --goal "test login"
skopix init
Interactive setup wizard. Saves API keys and tracker config to .skopix.env
skopix init
skopix report
Open the most recent session report in your default browser
skopix report
skopix report --dir ./other-reports
skopix config
View or modify your stored configuration values
skopix config --list
skopix config --set KEY=value
skopix config --get KEY
skopix suite
Run a saved suite of tests defined in a YAML file
skopix suite --file tests.suite.yaml
skopix --version
Print the installed Skopix version
skopix --version

Examples

Battle-tested goals you can use straight away
Tips for writing great goals
BE SPECIFIC
"log in with admin and dashboard, then click Reports in the sidebar" beats "test the app". The more specific your goal, the more reliable the test.
STATE THE EXPECTED OUTCOME
Always include what success looks like — "verify the order confirmation page appears with an order ID" — so the agent knows when to stop.
CHAIN STEPS WITH "THEN"
"Log in, then go to settings, then change the language to French, then verify the change saved." Skopix follows multi-step flows beautifully.
USE CREDENTIALS FILES
For login flows, save credentials in a YAML file. The agent will pick them up automatically without exposing them in the goal text.

Credentials

Securely store login credentials. Tests reference them by name — values stay safe in your home folder.
Credentials are stored in ~/.skopix/credentials.yaml with restricted file permissions. Tests reference them by name (e.g. panintelligence) — actual values never appear in suite YAMLs, logs, or screenshots.

Issues

All issues raised across your tests. Skopix de-duplicates against this list.
When a test detects an issue, Skopix records it locally with a fingerprint. On future detections of the same issue, Skopix comments on the existing tracker ticket instead of creating a duplicate. Resolved issues let new detections create fresh tickets (regression signal).

How to connect
Run skopix init in your terminal and follow the prompts. Or set the environment variables manually in your .skopix.env file.
# Interactive setup
skopix init

# Or set values directly
skopix config --set GITHUB_TOKEN=ghp_your_token
skopix config --set GITHUB_REPO=owner/repo
